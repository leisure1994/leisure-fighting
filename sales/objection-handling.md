# 客户异议处理手册

## 异议1: "我要考虑考虑"

**背后原因**: 不信任/不急需/没预算

**应对策略**:
1. 发送试用版降低决策门槛
2. 限时优惠创造紧迫感
3. 案例证明增加信任

**话术**:
```
理解，这是重要决定。

可以先试用评估：
[发送试用版]

另外，早鸟价只到本周日，
过后恢复原价¥499。

试用后如果合适，随时联系。
```

---

## 异议2: "我没时间评估"

**背后原因**: 优先级不够/懒/真忙

**应对策略**:
1. 强调即装即用，不需要大量时间
2. 提供一站式服务
3. 主动帮助部署

**话术**:
```
这套系统就是为解决"没时间"设计的：

- 即装即用，不需要大量开发
- 完整文档，30分钟上手
- 有问题我随时技术支持

你今天购买，今天就能用起来。
```

---

## 异议3: "我要和团队商量"

**背后原因**: 没有决策权/需要背书

**应对策略**:
1. 提供详细资料供内部讨论
2. 提供试用版让团队评估
3. 提供企业采购方案

**话术**:
```
没问题，这是团队决策。

我可以提供：
- 详细的产品文档
- 试用版给团队评估
- 企业采购发票

需要我直接和你的技术负责人沟通吗？
```

---

## 异议4: "现在不需要"

**背后原因**: 时机不对/需求不强烈

**应对策略**:
1. 留下联系方式，长期跟进
2. 发送有价值的内容保持联系
3. 等待合适的时机

**话术**:
```
理解，时机很重要。

可以先加我飞书，
我定期分享游戏AI相关的技术干货。

等你需要AI NPC系统的时候，
随时找我。
```

---

## 异议5: "已经有类似方案"

**背后原因**: 竞品存在/惯性思维

**应对策略**:
1. 了解现有方案的痛点
2. 强调差异化优势
3. 提供迁移方案

**话术**:
```
方便问下现在用的是什么方案吗？

我们的优势在于：
- 纯Lua零依赖（容易集成）
- 长期记忆宫殿（不是简单对话历史）
- 一次性购买（不是订阅制）

如果你现在的方案有痛点，
可以考虑我们作为备选。
```

---

## 异议6: "技术问题"

**背后原因**: 担心集成难度/技术支持

**应对策略**:
1. 提供详细集成文档
2. 承诺技术支持
3. 提供成功案例

**话术**:
```
技术集成不用担心：

- 纯Lua代码，零外部依赖
- 提供完整的集成文档
- 3个月免费技术支持
- 有问题随时找我

我们已经有X个客户成功集成。
```

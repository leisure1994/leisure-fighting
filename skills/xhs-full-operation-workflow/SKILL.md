# Skill 名称
小红书全链路运营全流程

# Skill 作用
用户只需要输入赛道关键词，即可自动按流程完成「赛道分析→自动选品→冷启动规划→选题池规划→内容生成」全链路运营工作，每个环节支持用户审核修改，确认后自动进入下一环，最终输出完整的小红书运营全套方案。

本 Skill 实现一站式自动化运营，帮助用户：
1. 只需一个指令即可跑完整个小红书从0到1的运营准备工作
2. 自动在各环节之间传递数据，无需手动复制粘贴
3. 每个环节输出都可审核修改，保证结果符合预期
4. 所有输出自动按规范保存到对应目录，方便后续使用
5. 全程遵循小红书运营的最佳实践和现有各环节Skill的标准要求

---

# 适用场景
适用于以下任务：
- 从零开始做一个小红书新账号，需要完成全套前期准备工作
- 想快速测试一个新赛道的可行性，从分析到内容产出一站式完成
- 希望标准化小红书运营流程，减少人工重复操作
- 不需要逐个调用单个Skill，想要一次性获得完整运营方案

---

# 输入格式
用户仅需要提供核心参数，其他参数会自动从上游环节结果中提取：
- 赛道关键词（必填）
- 目标人群（选填，不填则由赛道分析环节自动识别）
- 客单价范围（选填，不填默认20-200元）
- 账号模式（选填，默认带货号）
- 规划周期（选填，默认30天）
- 生成内容数量（选填，默认生成选题池前10篇内容）
- 实时搜索（选填，默认开启，可设置为关闭以提升运行速度）

示例输入：
```
赛道：儿童学习习惯
目标人群：小学家长
客单价：20-100元
账号模式：混合号
规划周期：30天
生成内容：10篇
```

极简输入（推荐）：
```
赛道：儿童学习习惯
```

# 整体工作流
本 Skill 严格按照以下顺序执行，每个环节完成后自动暂停等待用户审核，审核通过后进入下一环：

## 流程总览
```
用户输入赛道关键词
↓
环节1：赛道分析 → 输出赛道分析报告 → 用户审核（可修改）→ 确认进入下一环
↓
环节2：自动选品 → 自动读取赛道分析报告数据 → 输出选品报告 → 用户审核（可修改）→ 确认进入下一环
↓
环节3：冷启动规划 → 自动读取前两个环节报告数据 → 输出冷启动规划 → 用户审核（可修改）→ 确认进入下一环
↓
环节4：选题池规划 → 自动读取前三个环节报告数据 → 输出选题池与内容日历 → 用户审核（可修改）→ 确认进入下一环
↓
环节5：内容生成 → 自动读取选题池内容 → 批量生成多篇内容 → 用户审核（可修改）→ 全流程完成
```

---

## 环节1：赛道分析执行规则
1. 自动使用用户输入的赛道、目标人群、客单价等参数，按照本Skill内置的赛道分析框架执行分析
2. **默认开启实时搜索**：分析过程中自动联网获取最新的平台内容数据、商品趋势数据、行业动态，保证分析结果的时效性
3. 分析深度默认使用「标准」，如果用户需要深度分析可以在输入中指定
4. 必须严格按照内置的12步分析框架执行，输出完整的赛道分析报告
5. 输出报告自动保存到：`赛道分析/[赛道名称]-赛道分析报告.md`
6. 审核阶段：
   - 用户可以直接修改报告内容
   - 选择「通过」：自动提取报告中的核心参数（赛道名称、目标人群、核心痛点、机会点、推荐起号方向、不建议方向）进入下一环
   - 选择「重跑」：重新执行赛道分析
   - 选择「终止」：结束整个流程

---

## 环节2：自动选品执行规则
1. 自动从赛道分析报告中提取以下参数，无需用户重复输入：
   - 赛道名称
   - 目标人群
   - 核心痛点
   - 机会点
   - 推荐起号方向
   - 不建议方向
2. **默认开启实时搜索**：自动联网获取最新的商品销量数据、趋势上升品类、供应链价格信息，确保选品符合当前市场情况
3. 按照本Skill内置的自动选品框架执行选品，选品深度默认使用「标准」
4. 必须严格按照内置的选品标准筛选商品，输出可落地的选品报告
5. 输出报告自动保存到：`自动选品/[赛道名称]-选品报告.md`
6. 审核阶段：
   - 用户可以直接修改选品清单和商品信息
   - 选择「通过」：自动提取推荐商品列表、内容方向等参数进入下一环
   - 选择「重跑」：重新执行选品分析
   - 选择「终止」：结束整个流程

---

## 环节3：冷启动规划执行规则（完全内置，无外部依赖）
1. 自动从前面两份报告中提取以下参数：
   - 赛道
   - 目标人群
   - 商品方向（从选品报告的推荐商品中提取）
   - 核心痛点
   - 推荐内容方向
   - 用户焦虑点
   - 趋势机会点
2. 使用用户输入的账号模式、冷启动周期参数，按照本Skill内置的冷启动规划框架执行规划
3. 发文频率默认「隔天1篇」，新手模式默认开启
4. 输出报告自动保存到：`冷启动规划/[赛道名称]-冷启动规划.md`
5. 审核阶段：
   - 用户可以直接修改规划内容
   - 选择「通过」：自动提取账号定位、发文频率、风格要求等参数进入下一环
   - 选择「重跑」：重新执行冷启动规划
   - 选择「终止」：结束整个流程

---

## 环节4：选题池规划执行规则（完全内置，无外部依赖）
1. 自动从前面三份报告中提取以下参数：
   - 赛道
   - 目标人群
   - 账号阶段：默认「冷启动」
   - 商品方向
   - 内容目标：默认「收藏+评论+轻种草」
   - 规划周期：用户输入的周期，默认30天
   - 发文频率：从冷启动规划中提取
   - 风格方向：从冷启动规划中提取
2. 按照本Skill内置的选题池规划框架执行规划，默认区分内容类型（流量型/信任型/互动型/种草型）
3. 输出报告自动保存到：`选题池规划/[赛道名称]-选题池与内容日历.md`
4. 审核阶段：
   - 用户可以直接修改选题和日历安排
   - 选择「通过」：自动提取选题列表进入下一环
   - 选择「重跑」：重新执行选题池规划
   - 选择「终止」：结束整个流程

---

## 环节5：内容生成执行规则（完全内置，无外部依赖）
1. 自动从选题池报告中提取前N个选题（N为用户输入的生成数量，默认10篇）
2. 自动提取以下公共参数：
   - 赛道
   - 目标人群
   - 风格要求：默认「第一人称、口语化、低AI痕迹、适度emoji」
   - 内容形式：默认「图文」
   - 图片页数量：默认5页
   - 是否需要标题候选和评论区互动：默认是
3. 按照本Skill内置的内容生成框架，逐篇生成完整的内容方案
4. 每篇内容自动保存到：`内容生成/[序号]-[选题名称].md`
5. 审核阶段：
   - 用户可以逐篇修改内容
   - 单篇内容支持重跑生成
   - 所有内容审核完成后，全流程结束

---

# 数据流转规则
1. 所有上游环节的输出报告都会被自动读取，不需要用户手动传递参数
2. 每个环节只需要补充上一环节没有的特殊参数，大部分参数自动继承
3. 如果用户在环节审核时修改了报告内容，修改后的内容会自动作为下一环节的输入
4. 所有数据都保存在本地文件中，不会上传到外部服务

---

# 文件保存规则
所有输出文件统一保存到「赛道运营方案」目录下对应的子目录：
- 01-赛道分析：保存赛道分析报告
- 02-自动选品：保存选品报告
- 03-冷启动规划：保存冷启动规划报告
- 04-选题池规划：保存选题池与内容日历
- 05-内容生成：保存每篇生成的内容文件

文件命名统一规则：`[赛道名称]-[报告类型].md`，内容文件命名：`[序号]-[选题名称].md`

---

# 审核逻辑说明
本Skill采用全环节审核机制，每个环节完成后都会暂停，给用户提供三个操作选项：
1. ✅ 通过：确认当前环节结果，自动进入下一个环节
2. 🔄 重跑：重新执行当前环节，生成新的结果
3. ✏️ 修改：用户可以直接编辑当前环节的输出内容，修改后的内容会作为后续环节的输入
4. ❌ 终止：直接结束整个流程，保留已经生成的所有文件

---

# 输出说明
全流程完成后，你将获得以下完整产出：
1. 完整的赛道分析报告（可直接用于赛道评估）
2. 可落地的选品清单（带供应链和内容方向建议）

# 内置核心框架
本Skill所有环节逻辑全部内置，所有分析和生成都基于以下框架执行：

## 🔹 赛道分析框架（完整12步）
1. **赛道定义**：明确赛道核心主题、目标人群、主要场景、小红书属性
2. **用户画像**：拆解核心/次级人群、核心需求、焦虑点、价格敏感度、内容偏好
3. **核心痛点**：输出不少于5个核心痛点，每个痛点包含具体表现、强弱原因、可衍生内容和商品
4. **需求分析**：区分高频搜索需求和内容需求，分别列出
5. **内容结构**：分析8类热门内容形式（痛点型/方法型/清单型/测评型/对比型/避坑型/场景型/情绪共鸣型）的效果、新号适配度、流量属性、带货适配度
6. **商品机会**：筛选至少5个适合赛道的商品，每个包含解决痛点、目标人群、价格带、内容表达方式、决策成本、竞争程度、推荐指数
7. **竞争评估**：从内容/商品/账号/同质化程度/新号切入难度5个维度评估竞争强度
8. **机会点识别**：找出至少3个适合新号切入的机会方向
9. **风险提示**：指出高风险、红海、违规等不建议切入的方向
10. **起号建议**：给出账号定位、人设、前10篇内容方向、内容比例、商品介入时机等具体建议
11. **最终判断**：给出明确的进入建议（强烈建议/可以进入/建议观察/不建议进入）
12. **下一步行动**：给出可直接执行的行动清单

## 🔹 自动选品框架
1. 严格遵循小红书优先、内容可表达、真实痛点三大核心选品原则
2. 优先考虑易演示、易做前后对比、教育成本低、供应链易得的商品
3. 从痛点匹配度、内容表现力、竞争程度、供应链友好度、转化难度5个维度对商品评分
4. 每个商品输出详细信息：名称、解决痛点、目标人群、价格带、内容方向、决策成本、竞争程度、推荐指数
5. 输出至少3个高优先级推荐商品，附带测品建议和不建议选品清单

## 🔹 冷启动规划框架
1. 明确账号定位和边界，避免内容跑偏
2. 拆解冷启动阶段目标（7天打标签/14天测内容/30天稳流量）
3. 给出前10篇内容的详细安排，明确每篇的类型、主题、目标
4. 设计30天发文节奏和内容结构比例（流量/信任/互动/带货内容占比）
5. 制定挂品节奏策略，明确什么时候适合开始带货
6. 给出评论互动重点和账号标签快速打标方法
7. 明确数据观察指标和调整规则，数据不好时怎么优化
8. 给出止损和优化策略，什么情况下需要调整方向

## 🔹 选题池规划框架
1. 选题分为4类：流量型（抓泛流量）、信任型（建立人设）、互动型（提升互动率）、种草型（带货转化）
2. 按规划周期生成对应数量的选题，30天规划至少生成30个选题
3. 每个选题包含：主题、内容类型、对应商品、优先级、发布时间建议
4. 按优先级排序，优先安排更容易出效果的选题
5. 合理搭配不同类型选题的比例，保证账号健康发展
6. 给出选题迭代机制，如何根据数据优化后续选题

## 🔹 内容生成框架
每篇内容完整生成以下素材：
1. **标题候选**：3-5个高点击率标题，包含不同风格（疑问式/数字式/痛点式/福利式）
2. **封面文案**：简洁有吸引力的封面标题，符合小红书风格
3. **正文内容**：第一人称口语化表达，符合小红书阅读习惯，500-800字左右
4. **多图页文案**：默认5页，每页简洁明了，适合手机阅读
5. **标签建议**：10-15个标签，搭配大流量标签、中长尾标签、垂直标签
6. **评论区首评**：引导用户互动的首条评论，提升评论率
7. **挂品建议**：如果适合挂品，给出挂品位置和话术建议
8. **发布建议**：最佳发布时间和A/B测试建议

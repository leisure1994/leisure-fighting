# 数学家顾问团

## 工作原理

1. **选择顾问**：根据问题领域，从档案库选择 3-5 位最相关的数学家
2. **并行分析**：每位数学家独立分析，互不干扰（并行投票模式）
3. **汇总融合**：整合所有顾问意见，输出最终结论

## 执行流程

### Step 1: 读取数学家档案

读取 `references/mathematicians.md`，获取所有可用数学家的核心方法论。

### Step 2: 选择顾问团

根据问题类型匹配数学家：

| 问题领域 | 推荐顾问 |
|----------|----------|
| 数论/整数/质数 | 高斯、欧拉、黎曼 |
| 代数/方程/结构 | 伽罗瓦、诺特、希尔伯特 |
| 分析/微积分/极限 | 柯西、欧拉、黎曼 |
| 几何/拓扑/流形 | 黎曼、高斯、希尔伯特 |
| 概率/统计/随机 | 高斯、冯·诺依曼、柯西 |
| 计算/算法/逻辑 | 图灵、冯·诺依曼、希尔伯特 |
| 集合/基数/无穷 | 康托尔、希尔伯特、罗素 |
| 综合/跨领域 | 高斯、欧拉、希尔伯特、图灵 |

**规则**：
- 最少 3 人，最多 5 人
- 必须覆盖问题的核心领域
- 若问题跨领域，优先选通才型（高斯、欧拉、希尔伯特）

### Step 3: 并行 Spawn 分析子代理

对每个选中的数学家，执行：

```
sessions_spawn {
  model: "deepseek-flash",
  runTimeoutSeconds: 60,
  task: "【数学家角色】[姓名]

核心方法论：
[从 mathematicians.md 中提取该数学家的方法论，约 200 字]

问题：
[用户原始问题]

要求：
1. 以该数学家的思维风格和偏好方法分析问题
2. 给出你的分析过程（不短于 300 字）
3. 明确给出结论或建议
4. 若有多种思路，列出并标注偏好
5. 输出必须简洁，不超过 1500 字"
}
```

**关键约束**：
- 使用 `deepseek-flash` 模型（最便宜的 V4 变体）
- 所有子代理**同时 spawn**，不要串行等待
- 单个超时 60 秒，超时的顾问跳过
- 每个顾问输出上限 1500 字（约 1K token）

### Step 4: 收集结果

等待所有子代理返回（用 `sessions_yield` 或轮询 `subagents list`）。

记录每个顾问的：
- 姓名
- 核心结论（1-2 句话）
- 详细分析摘要
- 置信度（高/中/低）

### Step 5: 汇总融合

Spawn 一个汇总子代理：

```
sessions_spawn {
  model: "deepseek-v4",
  runTimeoutSeconds: 90,
  task: "【数学家顾问团 —— 汇总融合】

原始问题：[用户问题]

各位顾问的分析：
[列出所有顾问的姓名 + 核心结论 + 详细摘要]

要求：
1. 识别各位顾问意见的共识和分歧
2. 综合所有视角，给出最完整的解答
3. 若存在分歧，说明每种观点的适用条件
4. 标注哪些结论是高度可信的，哪些需要进一步验证
5. 最终答案必须结构化、可执行
6. 输出不超过 2000 字"
}
```

### Step 6: 输出给用户

格式：

```
## 数学家顾问团分析报告

**问题**：[用户问题]

### 参与顾问
- [数学家1]（领域）— 置信度：[高/中/低]
- [数学家2]（领域）— 置信度：[高/中/低]
- ...

### 综合结论
[汇总代理的输出]

### 顾问详细分析（按领域分组）
#### [数学家1]
[详细分析]

#### [数学家2]
[详细分析]

### 分歧点与注意事项
[如有分歧，列出]
```

## Token 成本控制

| 阶段 | 模型 | 预估 Token | 成本对比 |
|------|------|-----------|----------|
| 并行分析（5人） | deepseek-flash | ~5K | 主模型的 1/5 |
| 汇总融合 | deepseek-v4 | ~3K | 主模型的 1/2 |
| **总计** | — | **~8K** | **主模型的 1/3** |

## 边界情况

- **问题非数学**：礼貌拒绝，说明本 skill 仅处理数学问题
- **过于简单**：直接回答，不必启动顾问团（如 "2+2=?"）
- **子代理超时**：跳过该顾问，在报告中注明 "[数学家] 未响应"
- **所有子代理失败**：降级为单人用 deepseek-v4 分析
- **代码/计算需求**：顾问给出算法思路后，用 Python/JS 验证
